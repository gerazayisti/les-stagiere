# Security Policy

## Supported Versions

Notre plateforme maintient une sécurité sérieuse sur les données de ses candidats et entreprises partenaires. 
Seule la branche `main` active et les déploiements de production officiels font l'objet d'un audit continu.

## Signaler une faille / vulnérabilité

*Ne déclarez pas publiquement de faille de sécurité (via un Issue GitHub) avant que nous l'ayons corrigée.*

Si vous identifiez une brèche, qu'il s'agisse d'une exposition de variable ou d'un contournement d'accès Supabase :
1. Envoyez immédiatement un e-mail à l'administrateur du projet.
2. Incluez une courte description et idéalement une étape de reproduction (*Proof-of-Concept*).

Nous prendrons connaissance de tout message de sécurité sous 72h. Merci d'aider à maintenir une plateforme stable et fiable au Bénin !
